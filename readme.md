
## Instructions

The goal is to build a web app that allows users to search through iconic quotes from the French TV show Kaamelott.

**Main expected features:**

-   list of quotes
-   at least one text search
-   at least one filter by character (`characters.json`)
-   at least one sorting option (candidates are free to choose which field(s) to sort on)
-   pagination of results

**Data:**

-   quotes and characters are available in JSON format in the `assets` folder of the project

**Expected:**

-   the application can be hosted or run locally
-   a client and a server (frameworks like Next/Nuxt are accepted)
-   a complete Git history using the provided repository

**Rules:**

-   use Node.js on the backend and a frontend framework (Vue or React) — TypeScript is encouraged
-   we expect the project to work out of the box — if a database is needed, use a self-contained solution (e.g. SQLite, Docker)
-   avoid using third-party services (or provide us with the necessary API keys)
-   as a guideline, we expect this test to take approximately 4 hours

**About AI-assisted coding:**

Using AI tools (ChatGPT, Copilot, coding agents, etc.) is entirely optional — feel free to use them or not, depending on what you're comfortable with. If you do use AI to help implement parts of the test, please keep track of the prompts you used and be ready to discuss them. Keep in mind that if you rely on AI assistance, we may set higher expectations for the overall result.

**Points of attention (non-exhaustive):**

-   client/server communication via API
-   code quality (readability, structure, comments, etc.)
-   ease of installation, setup, and testing (Node version, package manager, dependencies)
-   adapt your level of effort to the role you're applying for — focus on areas that best showcase your expertise
-   include at least a few unit tests
